### 通用规范

#### 项目命名

    全部采用小写方式， 以中横线分隔。

    例：my-project-name


#### 目录命名

    参照项目命名规则；

    有复数结构时，要采用复数命名法。

    例：scripts, styles, images, data-models


#### HTML，JS，CSS文件命名

    参照项目命名规则。

    例：card-list.html card-list.js card-list.css

#### 缩进,空格，换行

    缩进使用2个空格；

    不要混用tab和space；

    不要在属性间隔等处使用多个tab或space；

    换行符统一用'LF'；

#### 字符编码

    约定一致采用无 BOM 的 UTF-8 编码。

#### 格式化工具

    规范中的代码格式（空格，空行，换行）等部分规范可通过格式化工具和编辑器设置来解决，但你需要了解这些规范。
